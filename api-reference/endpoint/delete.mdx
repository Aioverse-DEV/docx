---
title: 'Delete Resource'
api: 'DELETE https://api.example.com/v1/resources/{id}'
description: 'Delete a resource permanently'
---

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the resource to delete
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the deletion was successful
</ResponseField>

<ResponseField name="message" type="string">
  Confirmation message
</ResponseField>

<ResponseField name="deleted_id" type="string">
  The ID of the deleted resource
</ResponseField>

<Warning>
  This action is permanent and cannot be undone. The resource and all associated data will be permanently deleted.
</Warning>

<RequestExample>

```bash cURL
curl -X DELETE \
  https://api.example.com/v1/resources/res_1234567890 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch('https://api.example.com/v1/resources/res_1234567890', {
  method: 'DELETE',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

const data = await response.json();
```

```python Python
import requests

url = "https://api.example.com/v1/resources/res_1234567890"
headers = {
    "Authorization": "Bearer YOUR_API_KEY"
}

response = requests.delete(url, headers=headers)
data = response.json()
```

```go Go

    req, _ := http.NewRequest("DELETE", "https://api.example.com/v1/resources/res_1234567890", nil)
```

## Soft Delete Alternative

If you want to preserve the resource data while removing it from active use, consider updating its status to `archived` instead:

```bash
curl -X PATCH \
  https://api.example.com/v1/resources/res_1234567890 \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"status": "archived"}'
```

<Tip>
  Using `archived` status allows you to restore the resource later if needed
</Tip>
