## Welcome to the API

Our API provides a comprehensive set of endpoints for managing your documentation, content, and resources programmatically.

<Note>
  The API is RESTful and returns JSON responses. All requests must be authenticated.
</Note>

## Base URL

```
https://api.example.com/v1
```

## Authentication

Authenticate your API requests using Bearer tokens in the Authorization header:

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
  https://api.example.com/v1/endpoint
```

### Getting your API Key

<Steps>
  <Step title="Log in to your dashboard">
    Navigate to [dashboard.example.com](https://dashboard.example.com)
  </Step>
  <Step title="Go to API Settings">
    Find the API section in your account settings
  </Step>
  <Step title="Generate a new key">
    Click "Generate API Key" and copy it immediately
  </Step>
  <Step title="Store securely">
    Save your API key in a secure location - it won't be shown again
  </Step>
</Steps>

<Warning>
  Never share your API key or commit it to version control. Treat it like a password.
</Warning>

## Rate Limiting

API requests are rate-limited to ensure fair usage:

- **Free tier**: 100 requests per hour
- **Pro tier**: 1,000 requests per hour
- **Enterprise tier**: Custom limits

Rate limit information is included in response headers:

```
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1640995200
```

## Response Format

All API responses follow a consistent format:

### Success Response

```json
{
  "success": true,
  "data": {
    // Response data here
  },
  "meta": {
    "timestamp": "2024-01-01T12:00:00Z"
  }
}
```

### Error Response

```json
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable error message",
    "details": {}
  }
}
```

## HTTP Status Codes

| Status Code | Meaning |
|-------------|---------|
| 200 | Success - Request completed successfully |
| 201 | Created - Resource created successfully |
| 400 | Bad Request - Invalid request parameters |
| 401 | Unauthorized - Invalid or missing API key |
| 403 | Forbidden - Insufficient permissions |
| 404 | Not Found - Resource doesn't exist |
| 429 | Too Many Requests - Rate limit exceeded |
| 500 | Internal Server Error - Something went wrong |

## Pagination

List endpoints support pagination using cursor-based pagination:

```bash
GET /api/v1/resources?limit=20&cursor=abc123
```

### Parameters

- `limit` (optional): Number of items per page (default: 20, max: 100)
- `cursor` (optional): Pagination cursor from previous response

### Response

```json
{
  "success": true,
  "data": [...],
  "pagination": {
    "next_cursor": "xyz789",
    "has_more": true
  }
}
```

## Example Request

Here's a complete example using cURL:

```bash
curl -X GET \
  https://api.example.com/v1/documents \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

## Webhooks

Configure webhooks to receive real-time notifications about events:

```json
{
  "url": "https://your-domain.com/webhook",
  "events": ["document.created", "document.updated"],
  "secret": "your_webhook_secret"
}
```
